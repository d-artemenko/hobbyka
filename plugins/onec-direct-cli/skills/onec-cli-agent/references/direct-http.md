# Прямой HTTP-контракт 1С

## Цепочка

```text
Codex -> scripts/onec-cli.mjs -> опубликованный HTTP-сервис базы 1С
```

CLI не передаёт вопрос другой модели. Codex сам выбирает объекты, строит запрос 1С и отправляет только детерминированный вызов в базу.

## Endpoint

Для базы `http://server/BaseName` и корня `mcp`:

- `GET http://server/BaseName/hs/mcp/health`
- `POST http://server/BaseName/hs/mcp/rpc`

Сервис принимает Basic Auth. Опциональный код разрешения начала сеансов передаётся как `?uc=...` из `ONEC_HTTP_UNLOCK_CODE` и не печатается.

По умолчанию CLI также ищет локальный защищённый конфиг `~/.config/hobbyka/onec-direct.env`. Этот файл не входит в плагин и не должен передаваться вместе с ним.

## JSON-RPC

Список доступных функций:

```json
{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}
```

Прямой вызов функции внутри 1С:

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "execute_query",
    "arguments": {
      "query": "ВЫБРАТЬ ...",
      "maxRows": 50,
      "params": {"ИмяПараметра": "значение"}
    }
  }
}
```

Handshake `initialize` не нужен. SSE endpoint не используется.

## Разрешённые функции

- `list_metadata_objects`
- `get_metadata_structure`
- `execute_query`

Перед первым живым запросом выполнить `tools` и подтвердить фактическую схему аргументов. Переданное расширение регистрирует `query`, `maxRows` и необязательный объект `params`; лимит по коду расширения составляет от 1 до 2000 строк.

Значения `params` в переданной реализации ограничены строками, числами и булевыми значениями. Для дат использовать подтверждённую конструкцию языка запросов 1С либо отдельно согласованное серверное преобразование, не передавать объект даты наугад.

Не включать `ПЕРВЫЕ N` в текст: расширение само добавляет или ограничивает `ПЕРВЫЕ` по `maxRows`. В переданной версии строка разбора уже существующего `ПЕРВЫЕ` содержит ошибочную проверку символа; CLI поэтому блокирует ручную директиву и всегда использует безопасную ветку автоматической вставки.

## Источник контракта

Контракт подтверждён исходным кодом расширения 1С: HTTP-сервис `mcp_APIBackend` публикует `/health` и `/rpc`, контейнер метаданных регистрирует две metadata-функции, а `mcp_ВыполнениеЗапроса` регистрирует `execute_query`. Публичные исходники `vladimir-kharin/1c_mcp` подтверждают тот же прямой HTTP-маршрут.
