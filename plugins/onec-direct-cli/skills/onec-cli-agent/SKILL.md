# Direct 1C CLI

Работать по цепочке `Codex -> CLI -> HTTP-сервис внутри 1С`. Рассуждать и формировать запрос самостоятельно; не отправлять естественно-языковую задачу другой модели или агенту.

## Главный сценарий: сохраняемый рецепт отчёта

1. При первом подключении выполнить `node scripts/onec-cli.mjs config-check`, затем `health` и `tools`.
2. Сначала искать готовый рецепт в `reports/onec/<slug>/`. Если он решает ту же задачу, переиспользовать и при необходимости обновить его.
3. Для новой задачи скопировать [assets/report-recipe](assets/report-recipe) в `reports/onec/<slug>/`. Хранить там `build.py`, запросы `.q1c` и только подтверждённые таблицы соответствий; результаты складывать в `artifacts/` и не коммитить.
4. Сформулировать период, организацию и бизнес-фильтры. Подтвердить имена объектов и полей через `metadata-list` и `metadata-get`.
5. Создать по одному понятному `.q1c` на каждый источник и выполнить рецепт установленным Python с `openpyxl`:

```bash
python build.py --cli /path/to/onec-cli.mjs --output artifacts/report.xlsx
```

6. Добавить в `build.py` только проверки и преобразования конкретного отчёта: обязательные колонки, период, непустой ожидаемый результат, помесячные итоги и явные соответствия.
7. Сопоставлять сначала по устойчивым кодам, затем по точным нормализованным названиям. Нечёткое сопоставление применять последним и выводить его уверенность и непарные строки отдельно.
8. Проверить Excel и сохранить рабочий рецепт для следующего запуска. Не сохранять в Git выгруженные данные 1С.

Не повторять metadata discovery, если точные имена уже подтверждены живой 1С в этой же задаче.

## Поиск отчётов и доказательство ограничений

Перед выводом «объект не найден» или «получить невозможно» различить три сущности:

1. **Объект метаданных** — искать через `metadata-list` и `metadata-get`.
2. **Сохранённый вариант отчёта** — искать read-only запросом в подтверждённом метаданными `Справочник.ВариантыОтчетов`; длинное пользовательское название обычно не является именем метаданных.
3. **Уже сформированный результат** — проверить `tools`; отсутствие инструмента результата не означает отсутствие варианта отчёта.

Для поиска варианта сначала подтвердить структуру `Справочник.ВариантыОтчетов`, затем выполнить узкий запрос по `Наименование`, например:

```sql
ВЫБРАТЬ
    Варианты.Наименование,
    Варианты.КлючВарианта,
    Варианты.Отчет,
    Варианты.НаименованияПараметровИОтборов
ИЗ Справочник.ВариантыОтчетов КАК Варианты
ГДЕ Варианты.Наименование ПОДОБНО &Маска
```

Не считать `metadata-list` поиском по вариантам или данным. Его пустой ответ доказывает только отсутствие совпадения среди имён и синонимов метаданных выбранного типа. `ХранилищеЗначения`, возвращённое запросом как маркер типа, не доказывает чтение значений настроек.

Заявлять ограничение только после живой проверки каждого относящегося к нему слоя и приводить точную успешную пробу или ошибку. Если готовый результат недоступен, не останавливаться автоматически: когда пользователю подходит аналитическая реконструкция, подтвердить исходные регистры и собрать её рецептом. Явно назвать результат реконструкцией, а не точной копией варианта 1С. Останавливаться только когда пользователь требует точного совпадения с готовой формой, а инструмента чтения или формирования результата нет.

## Команды

```bash
node scripts/onec-cli.mjs config-check
node scripts/onec-cli.mjs health
node scripts/onec-cli.mjs tools
node scripts/onec-cli.mjs metadata-list --type Documents --name-mask Заказ --max-items 20
node scripts/onec-cli.mjs metadata-get --type Documents --name ЗаказКлиента
node scripts/onec-cli.mjs query --text "ВЫБРАТЬ ..." --max-rows 50
node scripts/onec-cli.mjs query --file /private/tmp/query-1c.txt --params-json '{"Организация":"Хоббика"}' --max-rows 50
node scripts/onec-cli.mjs query --file /private/tmp/query-1c.txt --max-rows 50
node scripts/onec-cli.mjs query --stdin --max-rows 50
node scripts/onec-cli.mjs query --file /private/tmp/query-1c.txt --dry-run
node scripts/onec-cli.mjs self-test
```

Для длинного запроса предпочесть `--file` или `--stdin`, чтобы текст не попадал в список процессов.

## Настройка

Хранить параметры в `~/.config/hobbyka/onec-direct.env`, `.env` рабочего каталога, `.env` навыка или отдельном файле через `ONEC_HTTP_ENV_FILE`. Общий файл в `~/.config/hobbyka/` позволяет одному защищённому конфигу работать и для установленного плагина, и для локальной копии навыка. Не просить пароль в чате.

```env
ONEC_HTTP_BASE_URL=http://server/BaseName
ONEC_HTTP_SERVICE_ROOT=mcp
ONEC_HTTP_USERNAME=ReadOnlyUser
ONEC_HTTP_PASSWORD=
ONEC_HTTP_CONNECT_IP=
ONEC_HTTP_TIMEOUT_MS=60000
```

Можно указать готовый адрес сервиса через `ONEC_HTTP_SERVICE_URL=http://server/BaseName/hs/mcp` вместо пары `BASE_URL` + `SERVICE_ROOT`.
Если DNS ещё указывает на внешний адрес, задать частный VPN-адрес через `ONEC_HTTP_CONNECT_IP`: CLI подключится к нему, сохранив проверку HTTPS по имени из `ONEC_HTTP_SERVICE_URL`.

## Жёсткие границы

- Использовать только `health`, `tools/list`, `list_metadata_objects`, `get_metadata_structure` и `execute_query` непосредственно в HTTP-сервисе 1С.
- Не запускать и не вызывать OpenCode, RAG, локальный MCP-сервер, SSE, Docker-прокси или стороннюю модель.
- Не выполнять произвольный tool по имени. CLI разрешает только две metadata-функции и `execute_query`, подтверждённые кодом переданного расширения.
- Принимать только запросы, начинающиеся с `ВЫБРАТЬ`, и отклонять команды изменения данных.
- Не писать `ПЕРВЫЕ N` вручную: передавать лимит через `--max-rows` от 1 до 2000. Это обходит дефект ветки разбора ручного `ПЕРВЫЕ` в переданном BSL-коде.
- Считать CLI-защиту дополнительной. Пользователь 1С обязан иметь read-only права на сервере.
- Разрешать обычный HTTP только для localhost, `.local` и частных IP; для другого HTTP-хоста требовать явный `ONEC_HTTP_ALLOW_INSECURE=1`.
- Не печатать пароль, Authorization header или unlock code.

Сегмент `/hs/mcp/rpc` и имена `tools/list`/`tools/call` заданы установленным расширением 1С. CLI отправляет JSON-RPC прямо туда без handshake и без промежуточного процесса. Детали: [references/direct-http.md](references/direct-http.md).
