# Terms

`onec-direct-cli` is provided as-is for authorized read-only access to a published 1C HTTP service. Use it only with a service and account you are permitted to access.

The user is responsible for enforcing read-only permissions in 1C, protecting credentials, validating query scope and results, and complying with applicable company policies and law. Client-side query checks are an additional safeguard, not a substitute for server-side access control.

Do not extend the allowed tool list or add data-changing operations without a separate security review.
