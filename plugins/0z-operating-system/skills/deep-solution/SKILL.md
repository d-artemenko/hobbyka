# Deep Solution

## Workflow

1. Read `AGENTS.md`, `tasks.md`, `ideas.md`, `STATUS.md`,
   `agent/status-map.md`, `agent/attention-map.md`, `agent/INDEX.md`,
   `agent/feedback.md`, and the relevant recent solutions.
2. Choose exactly one task for this run.
3. Check whether the task has a stopping condition: explicit `Done`, date,
   number of passes, acceptance rule, or a human decision.
4. If the task has no stopping condition, treat this as another pass, not as
   closure.
5. State why this task was chosen and what would make this pass useful.
6. For broad work, use narrow subagents where the current environment allows it.
7. Verify important claims yourself after subagent summaries.
8. Write the solution as thinking, not only as a summary: task, checked facts,
   findings, decision, risk, and next step.
9. Explain why the result does or does not resemble what the workspace owner
   would have chosen.
10. Update `STATUS.md` and `agent/INDEX.md`.

## Quality Bar

- Depth is more important than speed.
- Do not close an open-ended task because it feels good enough.
- If no obvious improvement is visible, widen the search: sources, edge cases,
  neighbouring risks, similar systems, and past mistakes.
- Do not edit `ideas.md` or `tasks.md` without explicit permission.
